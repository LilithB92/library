# 📚 Дипломный проект DF1: API для управления библиотекой

---

## 🖥️ Описание проекта

Цель дипломного проекта заключается в разработке REST API для управления библиотечной системой. Проект обеспечивает
управление книгами, авторами и пользователями библиотеки, а также позволяет отслеживать выдачу книг пользователям.

Проект реализован с применением современных технологий и подходов, обеспечивающих высокую производительность,
безопасность и удобство сопровождения.

## ⚙️ Установка и запуск проекта

Перед началом работы убедитесь, что у вас установлены Python, Docker и Docker Compose.

### Шаги по установке и запуску:

#### 1. Клонирование репозитория:

```bash
git clone https://github.com/LilithB92/library.git
```

#### 2. Переход в папку проекта:

```bash
cd PythonProject23
```

#### 3. Настройка переменных окружения:

Создайте `.env` файл в корневой директории проекта и добавьте необходимые переменные окружения:


```
cp env.example .env
```
Откройте созданный `.env` файл и установите значения переменных среды, соответствующие вашему окружению (например,
настройки БД, секретные ключи и прочее):
```
SECRET_KEY = your_secret_key_here
DEBUG = True
DB_NAME = your_db
DB_USER = postgres_user
DB_PASSWORD = postgres_password
DB_HOST = db
DB_PORT = 5432
```

#### 4. Запуск приложения с помощью Docker Compose:

```bash
docker-compose up -d --build
```

Это запустит контейнеры с базой данных PostgreSQL и вашим приложением Django.

## 📦 Система контроля версий

Проект использует систему контроля версий Git с моделью ветвления GitFlow. Основная работа ведется в ветке `develop`,
стабильные версии выпускаются в ветку `main`.

---

## 📑 Документация API

Автоматически генерируемая документация API реализована по стандарту OpenAPI. Доступна по адресу `/api/docs/` после
запуска приложения.

---

## 🛡️ Безопасность

Проект использует JWT-токены для аутентификации и авторизации пользователей. Все запросы к защищенным ресурсам требуют
наличия валидного токена в заголовке Authorization.

---

## 🎨 Оформление кода

Соблюдаются стандарты оформления кода PEP8. Используется автоматическое форматирование с помощью инструментов Black,
Flake8 и Isort.

---

## 📂 Хранение кода

Весь исходный код хранится в удалённом Git-репозитории для удобства совместной работы и истории изменений.
